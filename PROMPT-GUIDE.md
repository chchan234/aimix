# AI PORT 프롬프트 작성 가이드

> 이 문서는 AI PORT의 모든 AI 서비스 프롬프트 작성 시 반드시 참고해야 하는 가이드입니다.
> 프롬프트를 작성하거나 수정할 때 이 문서의 규칙을 따라야 합니다.

---

## 1. 핵심 원칙

### 원칙 1: 유료 서비스 품질
**"돈이 아깝다고 느끼면 안 된다"**

- 결과를 받았을 때 "와, 이 정도면 돈 낼 만하다"는 반응이 나와야 함
- 무료 서비스와 확실히 차별화되는 깊이와 퀄리티
- 전문가가 직접 분석한 것 같은 수준의 인사이트
- 개인화된 맞춤 분석 (일반적인 내용 X)

### 원칙 2: 자연스러운 표현
**"AI가 쓴 것처럼 보이면 안 된다"**

- 사람이 직접 쓴 것 같은 자연스러운 문체
- 딱딱하거나 기계적인 표현 금지
- 실제 전문가나 친구가 말하는 듯한 톤
- 한국어 맞춤법과 자연스러운 어순

### 원칙 3: SNS 공유 가능 수준
**"공유하고 싶을 만큼 깔끔하고 재밌어야 한다"**

- 친구에게 캡처해서 보내고 싶은 내용
- 인스타/카톡에 공유해도 부끄럽지 않은 퀄리티
- 재미있고 공감되는 포인트 포함
- 시각적으로 정리된 결과

### 원칙 4: 사용 욕구 자극
**"다시 사용하고 싶게 만들어야 한다"**

- 결과를 보고 다른 서비스도 써보고 싶게
- 친구에게 추천하고 싶은 경험
- "나도 해봐" 라고 말하게 만드는 퀄리티
- 중독성 있는 재미 요소

### 원칙 5: 디테일한 깊이
**"프롬프트도, 결과도 디테일하고 길어야 한다"**

- 프롬프트: 구체적인 지시사항, 예시, 형식 명시
- 결과: 최소 500자 이상의 충분한 분량
- 각 항목마다 구체적인 설명과 근거
- 피상적인 한 줄 분석 금지

## 2. 프롬프트 작성 규칙

### 2.1 필수 포함 요소

모든 프롬프트에 반드시 포함해야 할 요소:

```
1. 역할 설정
   - "당신은 10년 경력의 퍼스널컬러 전문가입니다"
   - "당신은 유명 피부과 전문의입니다"

2. 분석 항목 상세 명시
   - 분석해야 할 모든 항목을 구체적으로 나열
   - 각 항목의 세부 요소까지 명시

3. 결과 형식 지정
   - JSON 형식 또는 구조화된 형식 명시
   - 각 필드가 어떤 내용을 담아야 하는지 설명

4. 톤앤매너 지시
   - "친근하면서도 전문적인 톤으로"
   - "재미있지만 신뢰감 있게"

5. 분량 지시
   - "각 항목당 최소 2-3문장으로 상세히"
   - "전체 분석은 최소 500자 이상"

6. 금지 사항 명시
   - "일반적인 내용 금지"
   - "모호한 표현 금지"
```

### 2.2 프롬프트 길이 기준

| 서비스 유형 | 최소 프롬프트 길이 | 최소 결과 길이 |
|------------|------------------|---------------|
| 이미지 분석 | 800자 이상 | 500자 이상 |
| 성격 테스트 | 600자 이상 | 800자 이상 |
| 이미지 생성 | 500자 이상 | - |
| 텍스트 분석 | 600자 이상 | 600자 이상 |

### 2.3 결과물 구조

모든 결과는 다음 구조를 따라야 함:

```
1. 핵심 결과 (한눈에 보이는 메인 결과)
2. 상세 분석 (각 항목별 깊이 있는 설명)
3. 개인화된 조언/추천 (사용자 맞춤)
4. 재미 요소 (공유하고 싶은 포인트)
5. 추가 인사이트 (기대 이상의 정보)
```

## 3. 결과물 품질 기준

### 3.1 유료 서비스 수준 체크리스트

- [ ] 무료 서비스에서는 볼 수 없는 깊이인가?
- [ ] 전문가에게 상담받은 것 같은 느낌인가?
- [ ] 내 상황에 맞는 개인화된 내용인가?
- [ ] 새롭게 알게 된 정보가 3개 이상인가?
- [ ] 바로 실천할 수 있는 구체적인 조언이 있는가?

### 3.2 SNS 공유 가능 수준 체크리스트

- [ ] 캡처해서 친구에게 보내고 싶은가?
- [ ] 인스타 스토리에 올려도 될 만큼 예쁜가?
- [ ] 읽는 사람이 "나도 해보고 싶다"고 느끼는가?
- [ ] 웃기거나 공감되는 포인트가 있는가?
- [ ] 자랑하고 싶은 내용인가?

### 3.3 분량 품질 기준

**좋은 예:**
```
당신의 피부는 수분 보유력이 높은 편이지만, T존 부위의 피지 분비가
활발한 복합성 피부입니다. 특히 코 주변과 이마에서 유분기가 느껴지는
반면, 볼과 턱 라인은 건조해지기 쉬운 특징이 있어요. 이런 피부 타입은
계절에 따라 관리법을 다르게 가져가야 하는데, 여름에는 가벼운 수분
젤 타입을, 겨울에는 영양감 있는 크림을 부위별로 나눠 바르는 것을
추천드려요.
```

**나쁜 예:**
```
복합성 피부입니다. T존은 지성, 볼은 건성입니다.
```

## 4. 금지 사항

### 4.1 AI스러운 표현 금지 목록

절대 사용하면 안 되는 표현들:

```
- "~것으로 보입니다"
- "~것으로 판단됩니다"
- "~것으로 사료됩니다"
- "~할 수 있을 것입니다"
- "전반적으로 ~합니다"
- "종합적으로 볼 때"
- "다양한 측면에서"
- "여러 가지 요소를 고려했을 때"
- "~에 따르면"
- "일반적으로 ~"
- "통상적으로 ~"
```

### 4.2 피해야 할 패턴

```
1. 두루뭉술한 분석
   X: "좋은 피부를 가지고 계시네요"
   O: "피부결이 고르고 모공이 작은 편이에요. 특히 볼 부분의
      탄력이 좋아서 실제 나이보다 3-4살은 어려 보이실 거예요"

2. 일반론 나열
   X: "규칙적인 운동이 좋습니다"
   O: "당신의 체형은 하체 근력 운동이 특히 효과적이에요.
      스쿼트나 런지를 주 3회, 15회씩 3세트로 시작해보세요"

3. 감정 없는 나열
   X: "장점: A, B, C / 단점: D, E, F"
   O: "솔직히 말해서 이 조합은 꽤 희귀해요! A와 B가 동시에
      나타나는 경우가 10명 중 1명 정도거든요"

4. 과도한 긍정
   X: "모든 면에서 완벽합니다"
   O: "전체적으로 균형 잡힌 편이지만, 이 부분은 조금 더
      신경 쓰면 훨씬 좋아질 수 있어요"
```

### 4.3 톤앤매너 금지사항

```
- 너무 격식체 (보고서 같은 느낌)
- 너무 반말 (예의 없는 느낌)
- 과도한 이모지 남발
- 과도한 느낌표 (!!!)
- 빈 칭찬 ("대단해요!", "멋져요!")
```

## 5. 좋은 예시 vs 나쁜 예시

### 5.1 퍼스널컬러 분석

**나쁜 예시:**
```
당신은 웜톤입니다. 따뜻한 색상이 잘 어울립니다.
베이지, 브라운, 오렌지 계열을 추천합니다.
```

**좋은 예시:**
```
당신은 '가을 웜톤' 중에서도 특히 '딥 오텀' 타입이에요!

피부톤을 자세히 보면, 황색 베이스에 약간의 올리브 기운이
느껴지는데요. 이런 피부톤은 채도가 낮고 깊이감 있는 색상을
만나면 피부가 맑아 보이고, 눈이 또렷해지는 마법이 일어나요.

특히 당신에게 찰떡인 컬러는:
- 버건디 (와인 레드보다 조금 더 브라운 기운이 도는)
- 머스타드 (노란색이 아니라 겨자색이요!)
- 카키 (군복 색 말고, 올리브 그린에 가까운)

반면 피해야 할 컬러는 파스텔 핑크예요. 얼굴이 누렇게 떠 보일 수
있거든요. 혹시 핑크를 입고 싶다면 '더스티 로즈' 정도가 딱이에요!

💡 꿀팁: 립스틱 고를 때 '테라코타', '브릭레드' 키워드로
검색하면 당신에게 딱 맞는 색을 찾을 수 있을 거예요.
```

### 5.2 MBTI 분석

**나쁜 예시:**
```
당신은 ENFP입니다. 외향적이고 창의적인 성격입니다.
열정적이며 새로운 것을 좋아합니다.
```

**좋은 예시:**
```
ENFP - 재기발랄한 활동가

당신의 머릿속은 항상 새로운 아이디어로 가득 차 있죠?
아마 지금도 "이거 해볼까? 저것도 재밌겠다!" 하면서
여러 가지를 동시에 생각하고 있을 것 같아요.

🎯 당신의 숨겨진 강점
솔직히 ENFP는 '아이디어 뱅크'예요. 남들이 "그게 되겠어?"라고
할 때 "안 해봤잖아!"라고 말할 수 있는 용기가 있거든요.
이 특성 덕분에 주변에 당신과 있으면 "뭔가 될 것 같은"
에너지를 느끼는 사람이 많을 거예요.

⚡ 주의할 점
근데... 솔직히 말할게요. 시작은 창대하나 끝이 미약한 경우
많지 않았어요? 10개 시작해서 3개 끝내면 선방한 거죠.
이번 달 목표 중에 흐지부지된 거 분명 있을 거예요. (찔리면 손들기 🙋)

💼 찰떡 직업
콘텐츠 크리에이터, 스타트업 기획자, 여행 작가, UX 디자이너
(반복 업무 많은 회계사나 공무원은... 3개월 버티기 힘들 수도)

❤️ 연애 스타일
밀당? 그런 거 못해요. 좋으면 좋다고, 보고 싶으면 보고 싶다고
바로 말하는 타입. 연애 초반에는 상대방이 "와 나한테 이렇게까지?"
하다가 나중에 "좀 숨 막혀..." 할 수도 있으니 텐션 조절 필요해요.
```

## 6. 서비스별 프롬프트 체크리스트

### 작성 전 체크리스트

- [ ] 역할 설정이 구체적인가? (경력, 전문 분야 명시)
- [ ] 분석 항목이 5개 이상인가?
- [ ] 각 항목의 세부 요소가 명시되어 있는가?
- [ ] 결과 형식(JSON 등)이 지정되어 있는가?
- [ ] 톤앤매너가 명시되어 있는가?
- [ ] 분량 기준이 있는가?
- [ ] 금지 사항이 명시되어 있는가?
- [ ] 예시가 포함되어 있는가?

### 결과물 검수 체크리스트

- [ ] 500자 이상인가?
- [ ] AI스러운 표현이 없는가?
- [ ] 개인화된 내용인가? (일반론 X)
- [ ] 구체적인 수치/예시가 있는가?
- [ ] 재미 요소가 있는가?
- [ ] 실천 가능한 조언이 있는가?
- [ ] SNS에 공유해도 될 만큼 깔끔한가?
- [ ] 돈 낼 만한 가치가 느껴지는가?

## 7. 프롬프트 템플릿

### 기본 구조

```
당신은 [구체적인 역할/경력]입니다.

[입력 데이터]를 분석하여 다음 항목들을 상세히 분석해주세요:

1. [분석 항목 1]
   - 세부 요소 a
   - 세부 요소 b
   - 세부 요소 c

2. [분석 항목 2]
   ...

[결과 형식 지정]

[톤앤매너 지시]
- 친근하면서도 전문적인 톤
- 실제 전문가가 1:1 상담하는 것처럼
- 재미있는 비유나 예시 활용

[분량 지시]
- 각 항목당 최소 3문장 이상
- 전체 분석 500자 이상

[금지 사항]
- "~것으로 보입니다" 같은 AI스러운 표현 금지
- 일반적이고 누구에게나 해당되는 내용 금지
- 근거 없는 칭찬 금지

[추가 지시]
- 사용자가 SNS에 공유하고 싶을 만큼 재미있고 유용하게
- 돈 주고 받은 서비스라는 것을 잊지 않도록 퀄리티 유지
```
